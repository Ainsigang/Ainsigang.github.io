DHCP工作原理

发表于 2016-08-13

网络协议复习篇之二：DHCP的工作原理

概述

DHCP(Dynamic Host Configure Protocol),动态主机配置协议，用于向网络中的主机 自动分配 ip地址，避免了网络中出现IP地址冲突，同时大大降低了网络管理人员的工作负担。DHCP协议是一种应用层协议，运行与UDP协议之上。同时DHCP还是一种C/S架构协议，有客户端和服务端之分，请求IP地址的一端为客户端，以UDP端口号68进行数据传输，提供IP地址的一端为服务端，以UDP端口号67进行数据传输。

原理

在整个DHCP服务器为客户端分配IP的过程，一共经历了四个阶段，整个过程如图描述：

  • 发现阶段(DISCOVER)

在客户端配置了DHCP客户端程序并启动后，以 广播形式 向局域网内发送 DHCP DISCOVER 报文，以寻找局域网中的DHCP服务器。此广播报文在传输层采用UDP 68号端口发送。在网络层封装中，由于此时主机还没有可用的IP地址，故 源IP地址 为 0.0.0.0， 目的IP地址 为 255.255.255.255。在链路层中，局域网为以太网时， 源MAC地址 为主机的MAC地址， 目的MAC地址 为 FF-FF-FF-FF-FF-FF。

  • 提供阶段(OFFER)

局域网中的DHCP服务器接收到了DISCOVER报文之后，会根据自己IP地址池中IP分配的优先次序选出一个IP地址，然后与其他参数一起通过传输层的UDP 67号端口以广播方式发送给客户端，目的端口是UDP 68号。在网络层的封装中， 源IP地址 为 DHCP服务器的IP地址，目的IP地址为 255.255.255.255。在链路层的封装中， 源MAC地址 为 DHCP服务器的MAC地址。 目的MAC地址 为 客户端的MAC地址(从DISCOVER报文中获得)。

  • 请求阶段(REQUEST)

如果局域网中存在多台DHCP服务器，则客户端会收到多于一条的 DHCP OFFER报文，一般客户端会选择最先收到的一条。然后以广播的方式向这台DHCP服务器发送 DHCP REQUEST 报文。在传输层任仍然是 UDP 68号发送，目的端口为 UDP 67号，在网络层，由于此时客户端仍没有获得IP地址， 源IP地址还为0.0.0.0，目的IP地址为DHCP服务器的IP地址(从OFFER报文中获得)，在链路层，源MAC地址为客户端的MAC地址，目的MAC地址为DHCP服务器的MAC地址。

  • 确认阶段(ACK/NAK)

DHCP服务器确认给客户端分配IP地址的阶段。在DHCP服务器收到客户端的REQUEST报文后，如果确认将IP地址分配给客户端，则以广播方式回复客户端 ACK报文，否则回复NAK报文。在ACK报文中，源端口号为UDP 67，目的端口号UDP 68号。源IP地址为DHCP服务器的IP地址，目的IP地址为广播地址255.25.255.255。源MAC地址为DHCP服务器的MAC地址，目的MAC地址为客户端的MAC地址。同时在选项部分，会把所分配的IP地址的子网掩码、默认网关、DNS服务器、租约期、续约时间等信息加上。

最后，在客户端收到DHCP ACK报文后，会发送一个免费ARP报文来确认此IP地址在局域网内是唯一的。
