FTP协议原理

网络协议复习篇之三：FTP协议的原理

概述

FTP(File Transfer Protocol),文件传输协议。是internet中使用最广泛的文件传输协议，目的在于提高文件的共享性，提供非直接的远程操控计算机。FTP是一种应用层协议，建立在TCP协议之上，它需要建立两条TCP连接，一条用于数据传输，一般使用端口号21，一条用于连接控制，一般使用20号端口。

工作原理

使用FTP进行传输文件时，会使用两个TCP连接，第一个是FTP客户端与服务端的控制连接，FTP服务器开始TCP 21号端口，等待客户端发送连接请求，FTP客户端随机开启端口，向服务器发送连接请求，用于传输客户端与服务器端之间传输命令。第二个连接时数据连接，服务器端使用TCP 20号端口与客户端建立连接，通常情况下，服务器主动与客户端建立或断开数据连接。

FTP的传输模式

FTP传输数据时支持两种模式：ASCII和二进制模式。

  • ASCII模式

用于传输文本，发送端的字符在发送前将被转换成ASCII格式之后再进行传输，接收端接收后再将其转换成字符。

  • 二进制模式

常用于传输 图片文件和程序文件，发送端在发送时无需转换格式即可传输。

FTP的工作方式

FTP有两种工作方式：PORT方式和PASSIVE方式。

  • PORT方式

假定客户端用于建立控制连接的TCP 端口为2000，用于数据传输的TCP 端口为2001。

  1. 首先客户端使用TCP 2000号端口与服务器端21号端口建立一个TCP连接，即控制连接。
  2. 当用户发起一个要求发送或者接收文件的请求时，客户端软件使用PORT命令，这个命令包含了一个暂时端口(2001)，也包含了客户端自己的IP地址。
  3. 服务器端收到PORT命令后，以源端口20，目的端口为收到的PORT命令中的暂时端口(2001),与客户端经过三次握手建立一个TCP连接，即数据连接。(由服务端发起)
  4. 数据连接建立完成之后开始传输数据，以TCP进行数据传输保证了数据的可靠传输。当数据传输完成时，发送数据的主机发送一个FIN命令，这个FIN命令需要另一台主机的ACK确认，另一台主机也发送一个FIN名，同样需要发送数据的主机的ACK确认，然后完成数据传输，断开连接。
  • PASSIVE模式
  1. 同样客户端需要与服务器端建立一条TCP连接，用于控制连接，源端口号为2000，目的端口号为21.
  2. 当用户发起一个要求发送或者接收文件的请求时，客户端软件使用PASV命令给服务器端，命令包含客户端自己的IP地址和一个暂时端口(2001),表示希望进入PASSIVE模式。
  3. 服务器端收到PASV命令后对其进行应答，应答包含服务器的IP地址和一个暂时端口号(一个大于1024的端口号,不是20号)，这个暂时端口用于客户端建立数据传输连接。
  4. 客户端收到服务器端的应答之后，以源端口为2001，目的端口为收到的服务器端的应答中包含的暂时端口，与服务器端建立一条TCP连接，即数据连接。(由客户端发起)
  5. 数据连接建立完成之后开始传输数据，以TCP进行数据传输保证了数据的可靠传输。当数据传输完成时，发送数据的主机发送一个FIN命令，这个FIN命令需要另一台主机的ACK确认，另一台主机也发送一个FIN名，同样需要发送数据的主机的ACK确认，然后完成数据传输，断开连接。
